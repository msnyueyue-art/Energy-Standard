# 数据导出页面日期分隔符国际化修复说明

## 问题描述
在数据导出页面 (export-data.html) 的时间范围选择器中,两个日期输入框之间显示的分隔符"至"是硬编码的中文,在英文环境下没有切换为英文 "to"。

## 问题位置
- 文件: `export-data.html`
- 行号: 632
- 问题代码:
```html
<span class="date-separator">至</span>
```

## 修复方案

### 1. 添加国际化翻译键
在 `common.js` 中添加日期分隔符的翻译配置:

**中文翻译 (第3020行):**
```javascript
exportDateSeparator: '至',
```

**英文翻译 (第6760行):**
```javascript
exportDateSeparator: 'to',
```

### 2. 修改HTML元素
将硬编码的中文替换为带国际化属性的标签:

**修改前:**
```html
<span class="date-separator">至</span>
```

**修改后:**
```html
<span class="date-separator" data-translate="exportDateSeparator">至</span>
```

## 修复效果
- ✅ 中文环境: 显示"至"
- ✅ 英文环境: 显示"to"
- ✅ 语言切换时实时更新

## 测试方法
1. 打开数据导出页面: `export-data.html`
2. 观察时间范围选择器中两个日期输入框之间的分隔符
3. 切换语言(中/英文),验证分隔符是否正确切换:
   - 中文: 2026-01-07 16:54 **至** 2026-01-14 16:54
   - English: 2026-01-07 16:54 **to** 2026-01-14 16:54

## 相关文件
- `export-data.html` - 数据导出页面
- `common.js` - 国际化翻译配置文件
